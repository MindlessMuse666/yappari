# Техническое задание: Десктоп-приложение "Интервальное повторение (Японский → Русский)"

## 1. Общая концепция

Минималистичное десктоп-приложение для заучивания японских слов методом интервального повторения.  
**Платформа:** Windows 11 (нативная оболочка Wails + WebView2).  
**Языки:** японский → русский (только в одну сторону).  
**Пользователи:** однопользовательское, локальная БД SQLite.
**Название:** Yappari
**dev-название:** yappari
**Ключевые особенности:**

- Алгоритм SM-2 с ручной оценкой.
- Красивая фуригана через ручной ввод + фронтендовый рендеринг `<ruby>`.
- Автоматическая озвучка (Web Speech API) при показе ответа.
- Три режима тренировки: интервальное повторение, обычная зубрёжка, "ленивое заучивание".
- Тёмная тема ( #000000 / #ffffff / #c7cdd8 / #ff0a14 ).
- Полная доступность с клавиатуры и глобальные хоткеи.

## 3. База данных (SQLite)

### 3.1. Таблица `decks`

```sql
CREATE TABLE decks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 3.2. Таблица `cards`

```sql
CREATE TABLE cards (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    deck_id INTEGER NOT NULL,
    kanji_text TEXT NOT NULL,          -- Полное написание слова (например, "食べる")
    furigana_text TEXT,                -- Полное чтение каной (например, "たべる"), может быть NULL
    translation TEXT NOT NULL,         -- Русский перевод
    -- Поля SM-2
    ease_factor REAL NOT NULL DEFAULT 2.5,
    interval INTEGER NOT NULL DEFAULT 0,   -- дней
    repetitions INTEGER NOT NULL DEFAULT 0,
    next_review TEXT NOT NULL,             -- ISO8601 ("2026-07-10T10:00:00Z") или NULL, если ещё не изучалась
    last_review TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (deck_id) REFERENCES decks(id) ON DELETE CASCADE
);
```

**Правила валидации:**

- `kanji_text` обязательно.
- Если `furigana_text` == `kanji_text` или равно `NULL`, считаем, что слово не содержит кандзи (только кана). На фронтенде `<ruby>` не генерируем.
- `next_review` обновляется после каждого повторения в режиме интервального повторения. Для режима зубрёжки не меняется.

## 4. Бизнес-логика (Go Backend) и IPC

### 4.1. Экспортируемые методы (IPC)

В Go-структуре, которая биндится к Wails, реализуем:

| Метод | Параметры | Возврат | Описание |
|-------|-----------|---------|----------|
| `GetDecks` | – | `[]Deck` | Список всех колод |
| `CreateDeck` | `name string` | `Deck` | Создание новой колоды |
| `UpdateDeck` | `id int, name string` | – | Переименование |
| `DeleteDeck` | `id int` | – | Удаление колоды и всех её карточек |
| `GetCardsByDeck` | `deckID int` | `[]Card` | Все карточки колоды (без полей SM-2 кроме `next_review`?) – для управления |
| `CreateCard` | `card CardInput` | `Card` | Добавление карточки |
| `UpdateCard` | `id int, card CardInput` | – | Редактирование карточки |
| `DeleteCard` | `id int` | – | Удаление |
| `GetTrainingCards` | `mode string` ("interval"/"normal"/"lazy"), `deckIDs []int` | `[]TrainingCard` | Возвращает карточки для текущей сессии |
| `SubmitReview` | `cardID int, grade int` (0,3,4,5) | `Card` (обновлённая карточка) | Применяет SM-2 только в режиме "interval" |
| `ResetCardProgress` | `cardID int` | – | Сброс ease_factor, interval, repetitions, next_review = now |
| `ResetDeckProgress` | `deckID int` | – | Сброс всех карточек колоды |
| `CheckVoicesAvailability` | – | `VoiceStatus` (ja:bool, ru:bool) | Фронтенд может вызвать при старте |

**Особенности:**

- `GetTrainingCards` для `"interval"` выбирает только карточки с `next_review <= now()` из указанных колод. Для `"normal"` и `"lazy"` — все карточки из указанных колод (без проверки даты).
- Карточки возвращаются в случайном порядке (перемешиваются на бэке, чтобы фронт не зависел от порядка).
- `SubmitReview` обновляет поля SM-2 и `next_review`, `last_review`, `updated_at`. Валидирует, что `mode` был `"interval"` (фронт не должен вызывать для других режимов).

### 4.2. Алгоритм SM-2 (реализация в Go)

Вход: текущие `ease_factor`, `interval`, `repetitions`, `grade` (0,3,4,5).  
Выход: новые значения.

```text
if grade >= 3:
    if repetitions == 0:
        interval = 1
    elif repetitions == 1:
        interval = 6
    else:
        interval = round(interval * ease_factor)
    repetitions += 1
else:
    repetitions = 0
    interval = 1

ease_factor = ease_factor + (0.1 - (5 - grade) * (0.08 + (5 - grade) * 0.02))
if ease_factor < 1.3: ease_factor = 1.3

next_review = today + interval days
```

**Сброс прогресса:** устанавливает `ease_factor=2.5`, `interval=0`, `repetitions=0`, `next_review = now()` (чтобы карточка была доступна сразу), `last_review` очищается.

## 5. Frontend: компоненты и страницы

Используем Vue Router с тремя вьюхами:

### 5.1. Главная страница (`/`)

- Список всех колод с чекбоксами.
- Три большие кнопки:
  - "Интервальное повторение"
  - "Обычная зубрёжка"
  - "Ленивое заучивание" (доступна только если выбран режим зубрёжки)
- Чекбокс "Выбрать все" / снять выделение.
- Выбранное состояние чекбоксов сохраняется в `localStorage`.
- Кнопка "+ Новая колода" открывает модалку (PrimeVue Dialog) с полем имени.

### 5.2. Управление колодой (`/deck/:id`)

- Заголовок с именем колоды (редактируемый).
- Список карточек (таблица): японское слово, перевод, кнопки "Редактировать", "Удалить".
- Кнопка "Добавить карточку" → модальное окно с формой (см. ниже).
- Кнопка "Сбросить прогресс всей колоды".
- Возможность удалить колоду (с подтверждением).

### 5.3. Форма карточки (модальное окно, вызывается из любого места по Ctrl+N)

Поля:

- Японское слово (обязательное, `<input>`)
- Чтение (кана, опциональное)
- Перевод (обязательное)
- Выбор колоды (выпадающий список существующих колод, по умолчанию последняя созданная)

Валидация: слово и перевод не пустые.

### 5.4. Тренировка (`/training`)

- Заголовок с названием режима.
- Карточка:
  - Лицевая сторона: только `kanji_text` (без фуриганы).
  - Кнопка "Показать ответ" (или пробел).
  - После нажатия: отображается фуригана (рендеринг `<ruby>`), перевод, кнопки оценки (1,2,3,4) с подсказками "Повторить"/"Трудно"/"Хорошо"/"Легко" (только для интервального режима). В режиме зубрёжки/ленивом — только кнопка "Далее" (или стрелка вправо).
- Прогресс-бар (PrimeVue ProgressBar): `текущая_карточка / всего`.
- Кнопка-динамик для повторной озвучки (рядом со словом и переводом).
- По завершении — статистика (сколько карточек, время) и кнопка "Вернуться на главную".

## 6. Фуригана: frontend-рендеринг

На входе: `kanji_text` (например, `食べる`), `furigana_text` (`たべる`).  
Задача: выделить кандзи-части и проставить `<ruby>`.

**Алгоритм (упрощённый):**

- Если `furigana_text` отсутствует или полностью совпадает с `kanji_text` — показываем просто текст.
- Иначе разбиваем `kanji_text` на последовательности: кандзи-блоки и кана-хвосты.
- Для каждого кандзи-блока берём соответствующую часть фуриганы.
- Генерируем HTML:

  ```html
  <ruby>漢<rt>かん</rt></ruby><ruby>字<rt>じ</rt></ruby>食べる
  ```

  (окуригана остаётся снаружи `<ruby>`)
- Реализовать в виде Vue-компонента `FuriganaText`, который принимает два пропса.

Точную реализацию можно сделать через поиск границ кандзи в Unicode (\u4e00-\u9faf и т.д.) и сопоставление подстрок. Функция должна корректно обрабатывать слово `漢字` (кандзи-часть = "漢字", фуригана = "かんじ" → разбивка по символам).

## 7. Синтез речи (Web Speech API)

- Приложение при старте вызывает `CheckVoicesAvailability`; если отсутствуют `ja-JP` или `ru-RU`, выводит предупреждение с инструкцией.
- На тренировке:
  - По нажатию "Показать ответ":
    1. Прекратить текущую озвучку (если была).
    2. Произнести японское слово (`kanji_text` без фуриганы).
    3. Подождать 500 мс (настраиваемая пауза).
    4. Произнести русский перевод.
  - Кнопка "Динамик" повторяет эту же последовательность.
- В "ленивом режиме" озвучка происходит автоматически после имитации клика "Показать ответ".

**Технические детали:**

- Использовать `speechSynthesis.getVoices()` для поиска голосов. Убедиться, что они загружены (событие `voiceschanged`).
- Управление очередью: `speechSynthesis.cancel()` перед каждым новым воспроизведением.
- Пауза 500 мс реализуется через `setTimeout` перед вторым вызовом.

## 8. Режимы тренировки

### 8.1. Интервальное повторение

- Карточки загружаются через `GetTrainingCards("interval", selectedDeckIDs)`.
- Порядок случаен.
- Пользователь жмёт "Показать ответ", видит фуригану и перевод, слушает озвучку.
- Затем нажимает одну из 4 кнопок (хоткеи 1–4). Вызывается `SubmitReview`.
- Карточка уходит из стопки; после ответа на все показывается итог.

### 8.2. Обычная зубрёжка

- Загружаются все карточки выбранных колод (`GetTrainingCards("normal", ...)`).
- Случайный порядок.
- После показа ответа только кнопка "Далее" (или стрелка вправо) для перехода к следующей. Никакие SM-2 данные не меняются.
- Можно реализовать "бесконечный" цикл: после последней карточки начинаем сначала (перемешиваем заново). Но лучше сделать опционально (галочка "Зациклить").

### 8.3. Ленивое заучивание (суб-режим обычной зубрёжки)

- Тот же набор карточек, что и в обычной зубрёжке.
- Автоматический режим: без участия пользователя.
- Таймер (`setInterval`) эмулирует действия:
  1. Имитирует нажатие "Показать ответ".
  2. Ждёт завершения озвучки (JP → пауза 500 мс → RU) плюс дополнительные 2–3 секунды (настраиваемо).
  3. Автоматически переходит к следующей карточке.
- Кнопка "Пауза" для остановки таймера; переход в ручной режим возможен.
- При переходе в ручной – таймер сбрасывается, появляется кнопка "Далее".

**Настройка задержек:** можно вынести в конфигурационный файл (JSON) или хранить в `localStorage`.

## 9. Хоткеи и доступность с клавиатуры

- **Глобальные (работают поверх приложения):**
  - `Ctrl+N` — открыть форму создания новой карточки (модалка).
  - Внутри тренировочного вида:
    - `Space` — показать ответ / перейти к оценке.
    - `1` — Повторить (grade 0)
    - `2` — Трудно (3)
    - `3` — Хорошо (4)
    - `4` — Легко (5)
    - `Escape` — закрыть модалку, прервать тренировку (возврат на главную).
- **TAB-навигация:** Все интерактивные элементы (инпуты, кнопки, чекбоксы) должны быть доступны через `Tab` и `Shift+Tab`. PrimeVue обеспечивает это из коробки, но нужно проверить и при необходимости добавить кастомные обработчики на модалках.

**Шрифты:**

- Использовать системные шрифты: для русского — Segoe UI, для японского — Meiryo или Yu Gothic (Windows 11 предоставляет их). Указать в CSS `font-family: 'Segoe UI', 'Meiryo', 'Yu Gothic', sans-serif;`

**Компоненты PrimeVue:**

- Подключаем только нужные модули (Button, Checkbox, Dialog, InputText, ProgressBar, Select, DataTable, ...) через tree-shaking.
- Тему делаем либо через `unstyled` + свои классы, либо через кастомную тему, переопределив CSS-переменные PrimeVue на наши цвета.

## 11. Первый запуск и проверка окружения

1. При старте приложение проверяет наличие файла БД; при отсутствии — создаёт.
2. Вызывает `CheckVoicesAvailability` и при недостающих голосах показывает сообщение с инструкцией (как установить японский/русский языковые пакеты).
3. Главная страница пустая, предлагается создать первую колоду.

## 16. Финишные замечания для ИИ-агента

- **Никакого импорта/экспорта** — только ручной ввод.
- **Нет поиска/сортировки** слов.
- **Строго однопользовательский**.
- Все данные хранятся локально, синхронизации нет.
- Обратить внимание на безопасное завершение приложения: закрытие БД, остановка таймеров ленивого режима.
- Учесть, что Wails-сборка на Windows 11 может потребовать WebView2 Runtime (он предустановлен в Win 11, но стоит проверить).
